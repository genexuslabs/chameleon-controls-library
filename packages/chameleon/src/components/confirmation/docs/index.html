🎯 ch-confirmation Component Demo

Interactive demonstration of the ch-confirmation component with various states, styling examples, and event handling.

Example 1: Approval Requested State

This example shows the component in the approval-requested state with both Approve and Reject buttons visible.
<ch-confirmation
  state="approval-requested"
  title="Approval Required"
  request-message="Do you want to approve this document for publication?"
></ch-confirmation>

Example 2: Approval Responded (Approved)

This example shows the component in the approval-responded state after approval. No action buttons are shown.
<ch-confirmation
  state="approval-responded"
  title="✓ Approved"
  accepted-message="Your request has been approved..."
></ch-confirmation>

Example 3: Output Denied (Rejected)

This example shows the component in the output-denied state after rejection. No action buttons are shown.
<ch-confirmation
  state="output-denied"
  title="✗ Rejected"
  rejected-message="Your request was rejected..."
></ch-confirmation>

Example 5: Internationalization (i18n)

This example demonstrates how to customize button labels for different languages using the approve-button-label and reject-button-label properties.
<ch-confirmation
  state="approval-requested"
  title="Confirmación Requerida"
  request-message="¿Deseas aprobar esta acción?"
  approve-button-label="Aprobar"
  reject-button-label="Rechazar"
></ch-confirmation>

CSS Parts Reference

The component exposes the following CSS parts for styling:

Static Parts (always present):

  • ::part(container) - Main wrapper element (also includes state-specific parts)
  • ::part(title) - Title element (if title is provided)
  • ::part(message) - Message container
  • ::part(actions) - Actions container (only in approval-requested state)
  • ::part(button-approve) - Approve button (only in approval-requested state)
  • ::part(button-reject) - Reject button (only in approval-requested state)

State-specific Parts (on container):

  • ::part(container approval-requested) - When state is 'approval-requested'
  • ::part(container approval-responded) - When state is 'approval-responded'
  • ::part(container output-denied) - When state is 'output-denied'
  • ::part(container output-available) - When state is 'output-available'

CSS Custom Properties:

  • --ch-confirmation-gap - Gap between elements (default: 0.75rem)
  • --ch-confirmation-padding - Container padding (default: 1rem)
  • --ch-confirmation-actions-gap - Gap between buttons (default: 0.5rem)
